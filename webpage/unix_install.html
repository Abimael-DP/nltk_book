NLTK: Installing on Unix

Overview

The easiest way to install Python and NLTK is to use the RPMs provided on the NLTK CD. This should work out-of-the-box if you are running RedHat 7.1, or a sufficiently similar system. If you have trouble with using the RPMs, you might have better luck with either the SRPMs or with manual installation.

All the necessary files for each type of installation are provided in the unix subdirectory of the NLTK CD. If you do not have the NLTK CD, then the files can be downloaded from the following locations:

  • Python RPMs and SRPMs: http://www.python.org/2.2.1/rpms.html
  • Python tarball: http://www.python.org/ftp/python/2.2.1/Python-2.2.1.tgz
  • Numeric: http://www.python.org/2.2.1/rpms.html
  • NLTK: http://sourceforge.net/project/showfiles.php?group_id=30982

You will need to run all commands listed below as "root". Note also that the "%" character represents the command prompt, and should not be typed.

If you have already installed Python 2.1 or higher, or Numeric, then you do not need to re-install them.

Quick Start

For the impatient, the following set of commands will install everything you need. If none of the following commands give you errors, you're done. If they do, you'll need to read the rest of this document.

      rpm -i python2-2.1.1-3.i386.rpm 
      rpm -i python2-tkinter-2.1.1-3.i386.rpm
      rpm -i --replacefiles python2-tools-2.1.1-3.i386.rpm 
      rpm -i Numeric-20.1.0-1.i386.rpm
      echo Numeric >/usr/lib/python2.1/site-packages/Numeric.pth
      rpm -i nltk-1.0-1.noarch.rpm
      ln -s /usr/bin/python2.1 /usr/local/bin/python2.1
      ln -s python2.1 /usr/local/bin/python
      ln -s /usr/lib/python2.1 /usr/local/lib/python2.1
      ln -s /usr/include/python2.1 /usr/local/include/python2.1
      mv /usr/bin/python /usr/bin/python.bak
      ln -s python2.1 /usr/bin/python

Detailed Instructions

  • The RPM install instructions should be easiest, if you are running a compatible version of RedHat.
  • If you are not using RedHat 7.1, or if you have modified your system substantially since you installed it, then the provided RPMs may not work. If they don't, you can try the SRPM install instructions.
  • You can also install NLTK manually.
