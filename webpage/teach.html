Teaching with NLTK

NLTK provides ready-to-use courseware and a flexible framework for project work. Students augment and replace existing components, learn structured programming by example, and manipulate sophisticated models from the outset.

Assignments and Projects

NLTK can be used to create student assignments of varying difficulty and scope. In the simplest assignments, students experiment with an existing module. The wide variety of existing modules provide many opportunities for creating these simple assignments. Once students become more familiar with the toolkit, they can be asked to make minor changes or extensions to an existing module. A more challenging task is to develop a new module. Here, NLTK provides some useful starting points: predefined interfaces and data structures, and existing modules that implement the same interface.

Demonstrations

NLTK provides graphical tools that can be used in class demonstrations to help explain basic NLP concepts and algorithms. These interactive tools can be used to display relevant data structures and to show the step-by-step execution of algorithms. Both data structures and control flow can be easily modified during the demonstration, in response to questions from the class. Since these graphical tools are included with the toolkit, they can also be used by students. This allows students to experiment at home with the algorithms that they have seen presented in class.

See the NLTK quick start guide for information about how to run the demonstrations.

NLTK Lite

NLTK-Lite is a new collection of lightweight NLP modules designed for maximum simplicity and efficiency. NLTK-Lite only covers the simple variants of standard data structures and tasks. Simplicity and efficiency are valued over generality and extensibility.

NLTK-Lite is primarily intended to facilitate teaching NLP to students having limited programming experience. The focus is on teaching Python together with the help of NLP recipes, instead of teaching students to use a large set of specialized classes that lock them in to using NLTK.

Download NLTK-Lite

Contents:

  • corpora: brown.py, genesis.py, gutenberg.py treebank.py
  • draw: cfg.py plot.py tree.py
  • parse: cfg.py featurestructure.py parse.py tree.py
  • tag: brill.py hmm.py ngram.py unigram.py
  • tokenize: porter.py regexp.py stem.py
  • chat: eliza.py
  • doc: several tutorials

Key differences from NLTK:

  • requires Python 2.4
  • tokens are represented as strings, tuples, or trees
  • all tokenizers are iterators; large tasks produce output as early as possible
  • more emphasis on Python constructs instead of NLTK constructs
  • default pipeline processing paradigm leads to more transparent code
  • taggers incorporate backoff for smaller models and faster operation
  • shorter names (e.g. tokenizer.RegexpTokenizer() becomes tokenize.regexp())
  • tutorials are more easily maintained now with docutils and doctest
  • contributed software is more easily incorporated

Examples -- NLTK vs NLTK-Lite:

  1. Tokenization
    1. NLTK: 
          >>> from nltk.token import Token
    >>> from nltk.tokenizer import RegexpTokenizer
    >>> text_token = Token(TEXT = '''Hello.  Isn't this fun?''')
    >>> tokenizer = RegexpTokenizer(r'\w+|[^\w\s]+', SUBTOKENS='WORDS')
    >>> tokenizer.tokenize(text_token)
    >>> text_token
    <[, <.>, , <'>, , , , ]>
    2. NLTK-Lite: 
          >>> from nltk_lite import tokenize
    >>> text = '''Hello.  Isn't this fun?'''
    >>> list(tokenize.regexp(text, r'\w+|[^\w\s]+'))
    ['Hello', '.', 'Isn', "'", 't', 'this', 'fun', '?']
  2. Corpus access and tagging
    1. NLTK: 
          >>> from nltk.corpus import brown
    >>> items = list(brown.items())
    >>> train_tokens = []
    >>> num_words = 0
    >>> for item in items[:num_files*2/3]:
    ...     train_tokens.append(brown.read(item))
    >>> t0 = DefaultTagger('nn', SUBTOKENS='WORDS')
    >>> t1 = UnigramTagger(SUBTOKENS='WORDS')
    >>> for tok in train_tokens:
    ...     t1.train(tok)
    >>> t2 = NthOrderTagger(1, SUBTOKENS='WORDS')
    >>> for tok in train_tokens:
    ...     t2.train(tok)
    >>> del train_tokens
    >>> test_tokens = []
    >>> for item in items[num_files*2/3:num_files]:
    ...     test_tok = brown.read(item)
    ...     test_tokens.append(test_tok)
    >>> bigram = BackoffTagger([t2, t1, t0], SUBTOKENS='WORDS')
    >>> tagger_accuracy(bigram, test_tokens)
    .755
    2. NLTK-Lite: 
          >>> from nltk_lite.corpora import brown
    >>> from nltk_lite import tag
    >>> t0 = tag.Default('nn')
    >>> t1 = tag.Unigram(backoff=t0)
    >>> t2 = tag.Bigram(backoff=t1)
    >>> t1.train(brown.tagged('a'))
    >>> t2.train(brown.tagged('a'))
    >>> print tag.accuracy(t2, brown.tagged('b'))
    .755

Courses using NLTK

NLTK was designed for use in CIS-530 at the University of Pennsylvania, beginning in 2000. It has since been used in courses at several other institutions.

Please let us know if you teach natural language processing using NLTK, and we will add your course URL to this page. We will also subscribe you to the NLTK-Teach mailing list, where you can interact with others who use NLTK in their teaching.
